# C++ 对象声明、赋值及继承多态坑总结

## 1️⃣ Most Vexing Parse（最烦人的解析）

### 示例

```cpp
Person person();  // ⚠️ 不是对象，是函数声明
```

### 本质

* 被解析为：声明一个函数 `person`，无参数，返回 `Person`
* **不会**调用构造函数，也不会分配对象内存

### 风险

* 误以为创建了对象
* 不能调用成员函数：`person.show();` 会报错

### 避免方法

* 使用默认构造：

```cpp
Person person;     // 栈对象
Person person{};   // 统一初始化，避免 Most Vexing Parse
```

---

## 2️⃣ 栈对象（stack object）

### 示例

```cpp
Person person;
Person person = Person();  // 现代编译器通常优化为直接构造
```

### 特点

* 内存位置：栈
* 生命周期：作用域结束自动析构
* 构造函数调用：

  * `Person person;` 调用默认构造
  * `Person person = Person();` 可能调用拷贝构造，但现代编译器通常优化（拷贝省略）
* 调用方式：点号 `.`

### 优缺点

* ✅ 自动管理，无需手动释放
* ✅ 高效，缓存友好
* ❌ 生命周期固定，不能跨作用域长期存在

---

## 3️⃣ 堆对象（heap object / 动态分配）

### 示例

```cpp
Person *person = new Person();
delete person;  // 必须释放
```

### 特点

* 内存位置：堆（对象本身），指针在栈
* 生命周期：手动管理，需要 `delete`
* 调用方式：箭头 `->`

### 风险

* 内存泄漏、野指针、重复释放
* 现代 C++ 推荐使用智能指针：

```cpp
auto p = std::make_unique<Person>();
auto p = std::make_shared<Person>();
```

---

## 4️⃣ 对象赋值与切片（对象切片 Object Slicing）

### 示例

```cpp
TextFile tfile("text.txt", 1000, 2);
File file = tfile;  // 对象切片
```

### 现象

* `file` 只包含 **File 基类部分**
* `TextFile` 特有成员 (`chars`, `bytes`) 被丢弃
* 即使 `TextFile` 定义了 Getter，按值赋值也无法访问子类成员

### 解决方法

* 使用引用或指针 + 虚函数：

```cpp
TextFile tfile("text.txt", 1000, 2);
File& fileRef = tfile;       // 引用
cout << fileRef.fsize() << '\n';  // 调用 TextFile 中的 fsize()，多态生效
```

---

## 5️⃣ 继承与多态注意点

### 规则

1. 基类指针或引用可以指向子类对象（Liskov Substitution Principle）
2. 多态必须依赖 **虚函数 (`virtual`)**
3. 按值传递基类对象时，多态不生效，会发生对象切片

### 示例对比

```cpp
void printName(const File file);   // 按值传递 → 只复制基类部分
File& fileRef = tfile;             // 引用 → 多态生效（虚函数）
File* filePtr = &tfile;            // 指针 → 多态生效（虚函数）
```

## 6️⃣ 栈 vs 堆 vs 引用 vs 对象切片对比表

| 写法 | 本质 | 内存位置 | 构造函数 | 生命周期 | 调用方式 | 风险/注意 |
| --- | --- | --- | --- | --- | --- | --- |
| `Person person();` | 函数声明 | 无 | ❌ | 无 | ❌ | Most Vexing Parse，非对象 |
| `Person person;` | 栈对象 | 栈 | 默认构造 | 自动 | `.` | 安全，作用域结束自动析构 |
| `Person person = Person();` | 栈对象（拷贝/移动初始化） | 栈 | 默认构造，现代编译器拷贝省略 | 自动 | `.` | 旧编译器可能触发拷贝 |
| `Person* person = new Person();` | 堆对象 | 堆 | 默认构造 | 手动释放 | `->` | 需 delete，否则内存泄漏 |
| `File file = tfile;` | 栈对象切片 | 栈 | 默认构造 + 赋值 | 自动 | `.` | 子类成员丢失（对象切片） |
| `File& fileRef = tfile;` | 引用 | 栈指针，指向堆/栈 | 调用子类虚函数 | 随引用作用域 | `.` | 多态生效，安全访问子类方法 |
| `File* filePtr = &tfile;` | 指针 | 栈指针，指向堆/栈 | 调用子类虚函数 | 手动管理（指针） | `->` | 多态生效，需注意对象生命周期 |

---

## 7️⃣ 现代 C++ 最佳实践

1. **避免 Most Vexing Parse**：使用统一初始化 `{}`

   ```cpp
   Person person{};
   ```

2. **优先使用栈对象**，除非必须动态分配
3. **动态对象用智能指针管理**

   ```cpp
   auto p = std::make_unique<Person>();
   auto p = std::make_shared<Person>();
   ```

4. **多态必须引用或指针 + 虚函数**
5. **避免按值传递子类对象给基类**，防止对象切片

---

✅ 总结一句话：

* **`Person person();` → 函数声明**
* **栈对象 → 自动管理**
* **堆对象 → 手动管理或智能指针**
* **对象切片 → 按值赋值丢失子类信息**
* **多态 → 引用/指针 + 虚函数生效**
