# C++ 项目开发练习

---

## 项目概述

| 项目属性 | 内容 |
| -------- | ---- |
| **总项目名称** | 多线程图形化学生管理系统 (C++ 综合大作业 Pro 版) |
| **预计开发周期** | 1~2个月 (每天 2~4 小时)，按阶段推进，每周结束须提交 Git 记录与 README 更新 |

### 项目目标

通过一个完整、可扩展的实际项目，将《C++ 程序设计》课程所有核心章节知识点
 (基础、类与对象、继承与抽象类、异常处理、模板与 STL、多线程、GUI、综合应用)
一次性系统性掌握。

项目需同时支持 **命令行 (CLI)** 和 **图形界面 (GUI)** 两种交互方式，
最终实现一个功能完整、线程安全、异常健壮、代码结构清晰的"学生信息管理平台"。

### 总体要求

- 全部代码必须使用 **C++17** 标准编译 (`g++ -std=c++17`)。
- 必须使用 **Git** 进行版本控制，每天至少 1 次 commit，commit 信息须清晰，
  例如: `feat: 完成 Student 类与文件读写`。
- 每个大题完成后必须编写对应的**单元测试** (可使用 Google Test 或手动编写测试用例)。
- 最终提交物:
  - 完整源码
  - `CMakeLists.txt` (推荐使用 CMake 构建)
  - 详细 `README.md` (含功能演示截图、编译运行步骤、知识点覆盖说明)
  - 运行日志示例

## 🟢 大题一: 基础与文件操作

> **Kapitel 1 - Vorbereitung**

**目标:** 熟练掌握 C++ 基本输入输出、字符串处理、结构体/类基础、文件流操作 (`fstream`)、
格式化输入输出，以及异常前的基本错误处理逻辑，为后续所有模块提供坚实的数据存取基础。
要求所有操作必须支持中文姓名输入输出 (使用 `std::wstring` 或正确处理 UTF-8)。

---

### 1.1学生数据输入输出系统 (基础控制台交互)

✅ (已完成), 查看对应代码点[这里](20260405/main.cpp)

✅ GPT 5.3 优化版代码点[这里](20260405/main_optimized.cpp)

实现一个完整的控制台程序，要求用户循环输入多名学生信息 (姓名、年龄、成绩)，
每输入一条后立即在控制台以格式化表格形式输出当前所有已输入的学生信息。

**具体要求:**

- 姓名支持中文 (长度不超过 20 个字符)，年龄为 `0~150` 的整数，成绩为 `0.0~100.0` 的浮点数。
- 输入时需实时提示 `请输入第 X 位学生的姓名:`、`年龄:`、`成绩:`。
- 输出格式必须使用 `std::setw`、`std::left`/`right` 实现美观表格:
  - 列宽固定: 姓名 19、年龄 10、成绩 10
  - 成绩保留两位小数
- 支持输入 `exit` 退出输入循环。
- 必须在 `main` 函数中使用循环结构，不允许使用全局变量，所有数据暂存于 `vector` 中。

**测试要求:** 输入 5 条不同数据后退出，控制台必须完整显示所有信息且格式对齐。

## 🟡 大题二: 类与对象

> **Kapitel 2 - Klassen und Objekte**

**目标:** 掌握类的封装、构造函数、析构函数、成员函数、`const` 成员函数、友元、
运算符重载等面向对象核心概念，实现 `Student` 和 `StudentManager` 两个核心类。

---

### 2.1 设计 Student 类 (完整实体类)

**具体要求:**

- **私有成员:** `std::string name`、`int age`、`double grade`。
- **构造函数:** 至少提供 3 种——默认构造、带参构造、拷贝构造。
- **成员函数:**
  - `display()` (`const`，格式化输出单条学生信息)
  - `input()` (从 `cin` 读取并校验基本格式)
  - `getter`/`setter` (全部使用 `const` 正确修饰)
  - `operator<<` (友元重载，用于 `cout` 直接输出)
- 析构函数中打印 `Student 对象被销毁` 以便后续调试。
- 必须实现**深拷贝** (避免浅拷贝问题)。

**测试要求:** 创建 3 个不同 `Student` 对象，分别用 `display()` 和 `operator<<` 输出，
验证拷贝构造是否正确。

---

### 2.2 设计 StudentManager 类 (管理器类)

设计 `StudentManager` 类，私有成员使用 `std::vector<Student> students`。

必须实现以下 `public` 方法 (每个方法都必须有详细注释):

| 方法签名 | 说明 |
| -------- | ---- |
| `void addStudent()` | 调用 `Student::input()` 后加入 `vector` |
| `void deleteStudent(const std::string& name)` | 按姓名删除 (支持中文)，不存在时输出提示 |
| `void updateStudent(const std::string& name)` | 查找后允许修改年龄或成绩 |
| `void showAll()` | 以表格形式显示所有学生 |
| `size_t getCount() const` | 返回当前学生数量 |

**测试要求:** 完成增删改查全流程，至少操作 10 次，验证 `vector` 大小变化正确。

---

### 2.3 按成绩排序与搜索 (进阶)

在 `StudentManager` 中新增以下两个方法:

- `void sortByGrade(bool descending = true)`: 使用 `std::sort` 按成绩排序 (支持升序/降序)，
  并在注释中说明 `std::stable_sort` 与普通 `sort` 的区别。
- `Student* findByName(const std::string& name)`: 按姓名查找，返回指针 (不存在返回 `nullptr`)。

**测试要求:** 排序前后分别调用 `showAll()`，验证成绩顺序正确，
且姓名相同的学生相对顺序保持稳定。
